任务销毁需要做些什么？场景如下：
1.任务运行完成自动退出
2.任务运行发生异常，被系统强制结束
3.任务A正常运行过程中，任务B请求强制结束任务A
4.任务A正在等待一个信号量，任务B需要销毁这个任务
还有没有其他场景？

任务销毁的副作用：
1.当任务获得一个IPC对象信号后，执行过程中发生异常被销毁，如何处理？
  例如 MUTEX 被任务 A 锁住，但执行过程中访问非法内存地址，被系统强制销毁，任务 B 永远无法等到 MUTEX 解锁
2.当任务获得一个IPC对象信号后，执行过程中被其他任务销毁，如何处理？
  例如 MUTEX 被任务 A 锁住，但执行过程中，被另一个任务销毁，任务 B 永远无法等到 MUTEX 解锁
3.当任务等待一个IPC对象时(等待时间过长)，被其他任务销毁，如何处理？
  例如 MUTEX 被任务 A 锁住，任务 B 在等待 MUTEX 解锁，但是任务 A 长时间未解锁，此时服务程序认为 任务 B 等待时间过长，杀死任务 B.
